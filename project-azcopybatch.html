AZCopy and the Batch File

Leighton van Rooijen 31/08/2020

You would think it would be simple to turn the AZCopy script from the last post into a batch file. All I thought I would need to do was chuck that script into a text document and save it as a .bat file. Well it turns out there was one step that would have me running around the depths of the internet to work out. This elusive step had me almost giving up on AZCopy altogether, I explored every other option Azure had to offer but still people kept recommending AZCopy for a local backup.

It turns out that when you create a batch file you need to do a thing called escaping characters. Anyone who is familiar with any programming language will know certain characters are used for purposes other than the literal use. As the destination and source locations need to be in quotes for AZCopy I needed to use some escape characters. For example in batch files to have a single & character you actually need to type && as the & character is used as a joiner. You can find a full list of thesee by typing batch escape characters into google. But for now save yourself the headache and remember that these pesky little characters exist.

Once I had worked that out all I had to do was find and replace all of the characters with their escape characters and the file was up and running.
