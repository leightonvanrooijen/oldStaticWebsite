Using Task Scheduler to Automate Container Backup

Leighton van Rooijen 01/09/2020

After a grand total of 12 hours I decided I was sick of clicking on the AZCopy batch file I made to backup my website container to local storage. That’s when I went looking for methods to automate that double click. It turns out Windows has a built in program called Task Scheduler. Task Scheduler runs on Windows servers too so this method would work in a small production environment but this wouldn’t be in any way efficient if you had a lot of containers to back up. In that case I would use powershell. In all honesty I think I will migrate this task to Powershell in the future as it would be excellent ‘real world’ practice and it should only take a few minutes.

To begin I put my AZCopy batch file into my system32 folder so that there would be no permission issues to run the batch file and that’s where most automation batch files are typically placed. The next step was to boot up Task Scheduler and click create task. From here it’s as simple as following the steps. The UI is pretty intuitive so I had no problems doing this for the first time.

Simply name and describe the task so you can identify it from the hundreds of other tasks going on. Make sure you have a naming convention for your backup as if you get a few of these it will make management much simpler. Mine was azcopy-project-”nameofproject”-batch. Once named set the trigger. Strangely you can only select one trigger here but don’t worry you can edit the triggers and add as many as you want once you’ve created the task. I decided I wanted the task to run every day, gave it permission to do this while I wasn’t logged on, and to also run every time I logged into my account. I figured that currently I only edit the website files on this computer so logging in would mean I was likely to be editing the files. Now I needed to select the action to be performed each time the trigger was activated. In my case it was to run the batch file so the action I went with was to start a program. All to select the batch file to run and create the task.

Once the task was created I right clicked on it and went to properties to add the second trigger. From here I went to the trigger pane and simply clicked new and went through a similar process from before. To test this all worked I cleared all the files from the local backup folder where AZCopy was directed to sync the files to then logged off and back on. Opened up the Task Scheduler, looked at the task log and it had run so I checked the folder and the files were back in there.
